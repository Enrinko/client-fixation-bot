# Бот фиксации клиентов

[![CI](https://github.com/Enrinko/client-fixation-bot/actions/workflows/ci.yml/badge.svg)](https://github.com/Enrinko/client-fixation-bot/actions/workflows/ci.yml)

Мини-версия партнёрского бота риелторской компании (тестовое задание): риелтор
фиксирует клиента за собой через пошаговый диалог. CRM по условию задания заменена
локальной базой SQLite.

Живой бот: [@TestRealEstateRusBot](https://t.me/TestRealEstateRusBot)

## Что умеет

- `/start` → меню с кнопкой **«Зафиксировать клиента»**;
- пошаговый опрос: телефон клиента → телефон риелтора → ФИО клиента;
- телефоны нормализуются к виду `+7XXXXXXXXXX`: принимаются форматы `+7…`, `7…`, `8…`
  с любым кодом РФ и «голые» 10 цифр, начинающиеся с 9 (мобильные); пробелы, скобки
  и дефисы вычищаются; буквы, чужие коды и неверная длина отклоняются;
- заявка сохраняется в SQLite и бот отвечает «Клиент зафиксирован ✅»;
- защита от дублей: тот же телефон клиента второй раз → «Клиент уже в работе»,
  новая заявка не создаётся. Дубль ловится дважды: рано (на первом шаге, чтобы не
  гонять человека по всем вопросам) и атомарно (UNIQUE-ограничение в базе — гонка
  двух параллельных диалогов не создаст две заявки).

## Стек

Python 3.14 · aiogram 3.30 · SQLite (stdlib) · uv · pytest · ruff · Docker Compose · GitHub Actions

## Запуск (нужны только Docker и git)

```bash
git clone https://github.com/Enrinko/client-fixation-bot.git && cd client-fixation-bot
cp .env.example .env        # вписать BOT_TOKEN от @BotFather
docker compose up -d --build
```

Бот работает через long polling — публичный URL не нужен.
Логи: `docker compose logs -f bot`.

## Тесты и линт

```bash
docker compose run --rm bot uv run pytest       # или make test
docker compose run --rm bot uv run ruff check . # или make lint
```

## Архитектура

Слоёный монолит — осознанный выбор для задачи такого размера:

```
bot/handlers.py  — диалог (aiogram FSM), никакой бизнес-логики
bot/phone.py     — нормализация телефона, чистая функция
bot/storage.py   — SQLite-репозиторий, дедуп на UNIQUE-ограничении
bot/main.py      — сборка и запуск
```

28 тестов покрывают ядро: правила нормализации (22 кейса, включая мусор),
сохранение, дубли, персистентность между перезапусками.

## Осознанные упрощения

- **FSM-состояния в памяти** — перезапуск бота сбрасывает незавершённый диалог
  (готовые заявки в SQLite не теряются). Продакшен-вариант — RedisStorage.
- **Синхронный sqlite3 в асинхронном боте** — записи единичные и мгновенные;
  при реальной нагрузке — aiosqlite.
- **Bitrix не подключён** — по условию ТЗ вместо CRM локальная база.
- **Хендлеры без юнит-тестов** — они тонкие прокладки; ядро (валидация, дедуп)
  покрыто полностью.
